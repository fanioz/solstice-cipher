# Bug Report

## Summary
**Title**: Light beams fail to propagate reflections, splits, or translations from mirrors, splitters, filters, and portals
**ID**: BUG-0001
**Severity**: S1-Critical
**Priority**: P1-Immediate
**Status**: Closed
**Reported**: 2026-06-12
**Reporter**: Antigravity

## Classification
- **Category**: Gameplay
- **System**: Beam Propagation System (2D)
- **Frequency**: Always
- **Regression**: Yes (Broken during refactoring for Combiner nodes)

## Environment
- **Build**: main (uncommitted changes)
- **Platform**: macOS (All platforms affected)
- **Scene/Level**: All levels in the main game (Level 1, Level 2, Level 3)
- **Game State**: Any gameplay state where light beams interact with mirrors/splitters/filters/portals

## Reproduction Steps
**Preconditions**: Start the game on Level 1, 2, or 3.

1. Drag a mirror from the briefcase and drop it onto the board.
2. Rotate the mirror so that it intersects the main light beam coming from the Sun.
3. Observe the light beam hitting the mirror.

**Expected Result**: The light beam bounces off the mirror, rendering a reflected beam that can illuminate symbols or other mirrors.
**Actual Result**: The light beam terminates at the mirror. No reflected beam is drawn.

## Technical Context
- **Likely affected files**: [main.gd](file:///Users/fanioz/solstice-cipher/src/gameplay/main.gd)
- **Related systems**: Beam Propagation
- **Possible root cause**: In `src/gameplay/main.gd:calculate_light_rays()`, `active_rays` is cleared at the start of the pass, but any reflected/split/translated rays appended during the pass are overwritten and discarded by `active_rays = combiner_outputs` at the end of the pass.

## Evidence
- **Logs**: Code execution terminates propagation after one segment.
- **Visual**: The light beam hit-stops on mirrors with no reflection ray originating from the hit point.

## Related Issues
- None

## Notes
The propagation should process all instant reflections/splits/translations in a queue within the current pass, and only deferred combination outputs should seed the next pass.
